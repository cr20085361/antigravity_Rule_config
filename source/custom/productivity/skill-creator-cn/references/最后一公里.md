# 最后一公里：安装、激活、自测与打包

> 这份文档解决一个问题：**skill 做好了，怎么真正让它跑起来？**  
> 文件放错位置、会话没重启、测试用语太随意——这三点是初学者最易翻车处。跟着下面四步走，保准能用上。

---

## 第一步：把文件放对地方

**skill 是一个文件夹**，里面至少有一个 `SKILL.md` 文件。这个文件夹要放在正确的位置，Claude 才能认到它。

### 安装路径

| 类型 | 安装路径 | 适用场景 |
|------|----------|----------|
| **项目级**（只在当前项目里用） | `.claude/skills/<name>/` | 给这个项目专用的 skill |
| **用户级**（所有项目都能用） | `~/.claude/skills/<name>/` | 通用工具，到处都想用 |

**目录名必须和 `name` 字段保持一致。**

`name` 是什么？就是 `SKILL.md` 文件最顶上那段配置里写的名字，比如：

```
---
name: my-skill
description: ...
---
```

这段 `---` 包起来的配置叫做 **frontmatter**（文件头信息，告诉系统这个 skill 叫什么、干什么用）。如果 `name` 是 `my-skill`，那这个 skill 的文件夹就必须叫 `my-skill`，放在 `.claude/skills/my-skill/`。名字对不上，Claude 识别不了。

### 安装示例

假设你的 skill 叫 `invoice-helper`：

```bash
# 项目级安装（在项目根目录下操作）
mkdir -p .claude/skills/invoice-helper
cp -r /path/to/invoice-helper/* .claude/skills/invoice-helper/
```

---

## 第二步：确认装上了

**安装完之后必须重启会话**（关掉当前对话，重新开一个），Claude 才会重新扫描 skills 目录，把你的 skill 加进可用列表。

### 最简自查办法

新会话里，直接问 Claude：

> "你现在可以用哪些 skill？"

或者：

> "列出你能用的所有 skill。"

skill 出现在列表里说明安装成功。如果没有，依次检查：路径是否正确（`.claude/skills/<name>/`）、文件夹名和 `name` frontmatter 是否完全一致（含大小写）、`SKILL.md` 是否在文件夹里、会话是否真正**新开**（不是刷新页面）。

---

## 第三步：自测——skill 到底会不会被触发？

装上了不等于会被用。Claude 只在**任务足够复杂、值得调用专门 skill 的时候**才会触发它。如果你只是问一句"帮我读这个文件"，Claude 会直接用基础工具处理，不会去翻 skill。

**规律：任务越具体、越复杂、越有上下文，越容易触发。**

### 自测方法

在**新会话**里（这很重要，旧会话可能有缓存干扰），用接近真实需求的语句测试。下面是三个"好的测试语句"示例——可以直接拿来改着用：

**示例一（创建 skill）：**
> "我想为每天的晨会总结做一个 skill，输入是会议录音的文字稿，输出是带行动项的结构化摘要，帮我从需求讨论到写出 SKILL.md 完整走一遍。"

**示例二（改进 skill）：**
> "我有一个已有的 skill 叫 invoice-helper，但它生成的发票格式经常少字段。我想用测试用例跑一遍，看看问题出在哪，然后改进它。"

**示例三（优化触发描述）：**
> "我的 skill 做好了，但我觉得它触发率不高，能帮我优化一下 description 字段，让 Claude 在合适的时机更准确地调用它吗？"

### 如果 skill 没被触发

换一个更具体的说法，加上背景（文件名、业务场景、想要什么格式输出）。如果任务本身是"一句话就能直接做"的简单操作，它本来就不触发 skill——这是正常行为，不是 bug。

---

## 第四步：打包分发（可选）

如果你想把 skill 分享给别人用，或者备份一份方便迁移，可以把它打包成一个 `.skill` 文件。

**注意：这一步只在你有 `present_files` 工具的情况下才需要做。** 如果只是自己在项目里用，直接复制文件夹就够了。

### 打包命令

在 `skill-creator-cn` 目录下运行：

```bash
python -m scripts.package_skill <path/to/skill-folder>
```

例如，打包项目级的 `invoice-helper`：

```bash
python -m scripts.package_skill .claude/skills/invoice-helper
```

生成的 `.skill` 文件可以直接发给别人，对方解压放到自己的 skills 目录里就能用。

---

## 快速排查清单

| 问题 | 检查点 |
|------|--------|
| skill 没出现在列表 | 路径对不对？名字和 frontmatter 一致吗？新会话了吗？ |
| skill 出现了但从不触发 | 测试语句够复杂具体吗？description 写清了触发场景吗？ |
| 改了文件但没生效 | 必须重启会话（新开对话）才能重新加载 |
| 打包命令报错 | 确认在 skill-creator-cn 目录下运行，路径是 skill 文件夹 |
